# Echo Guardian v0.5 Native Runtime Implementation Start

v0.5 begins native runtime implementation while preserving the production laws established in v0.2-v0.4.

Implemented runtime starts:

- iOS Swift runtime bridge from minimized sensor observation to core-compatible JSON.
- Android Kotlin runtime bridge from minimized sensor observation to core-compatible JSON.
- Native audit append interfaces for local JSONL event output.
- Native policy/status screen models that block hidden behavior, cloud dependency, and default private-space monitoring.
- Native accessibility screen-flow skeleton for status, confirmation, and alert preview.
- Guarded provider test harness remains behind minimal-disclosure and authorization rules.

Production laws still enforced:

- No cloud dependency for the core safety loop.
- No raw sensor retention by default.
- No private-space monitoring by default.
- No hidden telemetry.
- No emergency-services dispatch by default.
- No live escalation outside guarded policy.
