# Echo Guardian v0.7 Device Execution + Signed Native Export

v0.7 starts device-execution hardening for native iOS and Android paths. It does not approve clinical deployment, medical-device claims, emergency-system certification claims, HIPAA claims, GDPR/global-compliance claims, or silent enterprise/MDM control.

## Implemented scope

- Physical-device execution checklist.
- Native persisted signing proof contract.
- Signed native export manifest contract.
- Native audit checkpoint signing contract.
- Provider sandbox receipt verification.
- Native accessibility UI test artifacts.
- CI artifact attestation baseline.
- Release-blocking Android native test verification requirement.

## Production laws preserved

- No hidden behavior.
- No private-space monitoring by default.
- No raw sensor retention by default.
- No automatic export.
- No emergency-services route by default.
- No cloud dependency for the core safety loop.
- Every native state transition that changes safety state must be auditable.

## Device execution checklist

A physical-device run is not considered ready unless all required evidence flags pass:

1. physical device target selected
2. permission/status screen visible
3. private-space monitoring blocked
4. raw-data retention blocked
5. native audit append enabled
6. device signing key persisted
7. export requires user confirmation
8. provider sandbox receipts verified
9. accessibility flow tested

## Signed native export

The native signed export manifest signs only user-confirmed export manifests. The signer blocks manifests where `user_confirmed` is not true or `automatic_export` is not false.
