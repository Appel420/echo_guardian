# Echo Guardian v0.8 Physical Device Evidence + Release Attestation

v0.8 adds a real-device evidence format and local release-attestation bundle for native iOS and Android validation.

This release remains a production foundation. It is not approved for HIPAA claims, GDPR/global-compliance claims, medical-device claims, emergency-system certification claims, clinical deployment, or silent enterprise/MDM control.

## Evidence rules

- Evidence must identify a physical device target.
- Simulator/emulator evidence cannot satisfy the real-device gate.
- Permission/status visibility must be captured.
- Private-space blocking must be captured.
- Raw-data blocking must be captured.
- Signed audit checkpoints must verify before evidence is bundled.
- Native export manifests must be user-confirmed and signed.
- Provider sandbox transcript evidence must be redacted.
- Accessibility evidence must include a visible or spoken “What are you doing?” path.

## Release attestation

The CI release attestation bundle hashes release artifacts, records native evidence bundle hash, requires the Android Gradle wrapper path, and treats Android native tests as release-blocking.
