# Echo Guardian v0.4 Native Implementation Start

Status: implementation scaffold, not clinical certification.

v0.4 starts native platform implementation while preserving the product laws:

- real local device signals
- no hidden behavior
- no private-space monitoring by default
- no raw sensor retention by default
- no cloud dependency for the core safety loop
- no emergency-services path by default
- no compliance, medical-device, or global-certification claims

## Native targets

- iOS Swift Package: `native/ios/EchoGuardianKit`
- Android Kotlin module: `native/android/echo-guardian-core`

## v0.4 native scope

- device-bound proof-of-possession signing prototypes
- minimized native sensor observation records
- plain-language status screen models
- guarded local-outbox provider boundary
- CI job separation for Python, iOS, Android, security, and SBOM

## Not approved in v0.4

- HIPAA compliance claims
- GDPR/global-compliance claims
- medical-device claims
- emergency-system certification claims
- clinical deployment
- silent enterprise/MDM control
