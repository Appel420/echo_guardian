# v0.7 Provider Sandbox Delivery Receipt Verification

v0.7 requires provider sandbox receipt verification before guarded provider delivery can advance toward live use.

A valid receipt must:

- come from sandbox mode
- match the expected provider
- match the expected provider message identifier
- represent an `attempted` or `delivered` status
- exclude sensitive details
- exclude emergency-services routing

Provider receipt verification is not approval for emergency dispatch or clinical deployment.
