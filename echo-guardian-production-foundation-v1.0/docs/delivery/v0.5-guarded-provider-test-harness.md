# Echo Guardian v0.5 Guarded Contact Delivery Provider Test Harness

v0.5 preserves the guarded provider boundary.

The provider harness must block:

- unauthorized contacts
- emergency-services routing by default
- sensitive-detail payloads
- delivery outside configured policy

Allowed provider behavior:

- prepare a minimal-disclosure alert for an authorized contact
- return delivery status for attempted, blocked, delivered, failed, or retrying states
- require audit entries for every state transition

The native provider harness does not create a hidden communications path. Production provider integrations must preserve the same interface and guardrails.
