# v0.4 Guarded Contact Delivery Provider Implementation

The provider boundary supports live contact delivery without weakening escalation rules.

## Rules

- Contacts must be authorized before delivery.
- Minimal disclosure is mandatory.
- Sensitive details are blocked by default.
- Emergency-services routing is blocked by default.
- Every attempted, blocked, failed, delivered, retrying, or cancelled delivery must produce an audit entry.

## Current provider

`LocalOutboxDeliveryProvider` is implemented for safe integration. It creates a local attempted-delivery state and does not perform external network delivery. Real SMS, email, push, or voice-call providers must implement the same boundary and pass guardrail tests before activation.
