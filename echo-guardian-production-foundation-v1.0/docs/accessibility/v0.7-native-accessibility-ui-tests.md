# v0.7 Native Accessibility UI Tests

v0.7 adds native accessibility test requirements for iOS and Android screen flows.

Required flows:

- local status screen
- permission/status screen
- “What are you doing?” response
- confirmation screen
- alert preview
- export confirmation
- delivery receipt review

Plain-language content must remain understandable to children and elderly users. A nonresponse must not be treated as consent, safety, or dismissal.
