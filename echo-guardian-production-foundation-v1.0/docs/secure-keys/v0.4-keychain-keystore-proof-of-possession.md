# v0.4 Keychain / Keystore Proof-of-Possession Prototype

## Purpose

Prove that a local device holds a device-bound signing key without exporting the private key.

## iOS

The Swift package provides `DeviceSigningKeyManager` using CryptoKit Curve25519 signing keys stored in Keychain with `ThisDeviceOnly` accessibility. Secure Enclave must only be used for operations the platform actually supports; unsupported primitives must not be represented as Secure-Enclave-backed.

## Android

The Kotlin module provides `DeviceSigningKeyManager` using Android Keystore with ECDSA P-256 proof-of-possession signing. StrongBox can be selected later when available and policy-approved.

## Portable development profile

Python tests use a software-protected Ed25519 development key. This is only for local validation and must not be described as hardware-backed.

## Required audit events in production app

- signing key created
- proof-of-possession generated
- key unavailable
- key reset or rotated
- trust boundary changed
