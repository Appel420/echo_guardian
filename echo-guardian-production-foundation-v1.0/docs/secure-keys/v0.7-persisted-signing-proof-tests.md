# v0.7 Persisted Signing Proof Tests

v0.7 adds a persisted signing proof contract for iOS Keychain and Android Keystore integration.

The Python reference implementation persists a development Ed25519 seed only for CI/local validation. Native platform implementations must map the same proof contract to platform stores:

- iOS: Keychain-backed persisted signing key; Secure Enclave only where the selected primitive is actually supported.
- Android: Android Keystore-backed signing key; StrongBox where available and policy-approved.

The tests verify:

- a non-empty challenge is required
- the public key remains stable after reloading the key store
- proof signatures verify
- native protection labels do not overclaim hardware protection
