# Echo Guardian v0.5 Keychain / Keystore Persisted Key Tests

v0.5 adds native persisted-key test scaffolds and proof-of-possession interfaces.

## iOS

The Swift package includes `DeviceSigningKeyManager`, which creates or loads a device signing key from Keychain and signs a challenge.

The implementation deliberately labels protection as `keychain_or_supported_secure_enclave_profile` because Secure Enclave support depends on the specific algorithm and platform profile. The build must not claim Secure Enclave protection for unsupported primitives.

## Android

The Kotlin module includes `DeviceSigningKeyManager`, which creates or loads an Android Keystore signing key and signs a challenge.

The implementation labels protection as `android_keystore_or_strongbox_when_available` because hardware-backed StrongBox availability depends on device capability.

## Rule

Device-bound proof-of-possession can be used for audit checkpoints, export manifests, and future managed policy attestation. It is not a medical, emergency-system, HIPAA, or GDPR compliance claim.
